<h1 aligin="center">Спецификация <code>BDSF</code> 0.1</h1>
<h2><strong>1.</strong> Формат содержимого файла</h2>
<h3><strong>1.1.</strong> Формат хранения данных</h3>
Все данные имеют следующий формат:

```py
KTV

Где K - это значение любого типа (формат записи: TC), которое обозначает ключ следующих данных,
  а T - это 1 байт типа данных,
  а V - это сами данные, записаные по формату, указаного в таблице.
```

<h3><strong>1.2.</strong> Базовые типы</h3>
В этом пункте описаны самые базовые типы.

```
Тип данных ———————————————— Длина(Байты : Биты) ———— Диапозон ———————————————— Код типа —— Формат
Byte                        1        :        8      0      - 255              01          -
Int16                       2        :       16      -2¹⁵-1 - 2¹⁵+1            02          Стандартный "big" HEX формат
Int32                       4        :       32      -2³¹-1 - 2³¹+1            03          Стандартный "big" HEX формат
Int64                       8        :       64      -2⁶³-1 - 2⁶³+1            04          Стандартный "big" HEX формат
UInt16                      2        :       16      0      - 2¹⁶-1            05          Стандартный "big" HEX формат
UInt32                      4        :       32      0      - 2³²-1            06          Стандартный "big" HEX формат
UInt64                      8        :       64      0      - 2⁶⁴-1            07          Стандартный "big" HEX формат
Float                       4        :       32      1      - 7 a.             08          IEEE 754-2008
Double                      8        :       64      8      - 15 a.            09          IEEE 754-2008
Decimal128                  16       :      128      16     - 23 a.            0A          IEEE 754-2008
Boolean                     1        :        8      0      - 1                0B          C{1 : Значение 0 или 1 (Аналогия с C++: 0 = false; 1 = true)}
String                      ?        :        ?      -                         0C          L{2 : Длина кодированной строки ~UInt16}
                                                                                           C{L : Строка в UTF-8}

# a. - Знаки после точки
# Не описаные ранее форматы описываются в следующем виде: <char>{<len> : <description> [~<type>]}...
# : <...> - Обязательно; [...] - Не обязательно
# - char:        Символ, обозначающий описание байты
# - len:         Длина в байтах, может быть символом (см. char), так же может содержать N, который указывает на длину элементов в сумме
# - description: Описание байтов
# - type:        Ранее описанный тип
```

<h2><strong>2.</strong> Примеры</h2>
<h3><strong>2.1.</strong> "Hello, World!"</h3>

```js
JSON ——————————————————————————— Бинарное представление (BDSF) —————————————————— Комментарии
{"hello": "world"}               0C 00 05 68 65 6C 6C 6F                          // Ключ (здесь - строка из 5 байт "hello")
                                 0C                                               // Тип последующих данных (здесь - String)
                                 00 05 77 6F 72 6C 64                             // Значение (данные, здесь строка из 5 байт "world")
```

Давайте сравним `BDSF`, `BSON` и `JSON`.
BDSF      | BSON       | JSON      | Разница
--------- | ---------- | --------- | --------------
`16` байт | `22` байта | `17` байт | `6` и `1` байт

<h3><strong>2.2.</strong> Расширенный пример</h3>

```js
JSON ——————————————————————————— Бинарное представление (BDSF) —————————————————— Комментарии
{
    "number": 1,                 0C 00 06 6E 75 6D 62 65 72 05 00 01              // Ключ - String; Данные - UInt16
    "float": 0.1,                0C 00 05 66 6C 6F 61 74 08 CD CC CC 3D           // Ключ - String; Данные - Float
    "boolean": true,             0C 00 07 62 6F 6F 6C 65 61 6E 0B 01              // Ключ - String; Данные - Boolean
    "string": "Hello, World!",   0C 00 06 73 74 72 69 6E 67                       // Ключ - String
                                 0C 00 0D 48 65 6C 6C 6F 2C 20 57 6F 72 6C 64 21  // Данные - String
    "list": [                    0C 00 04 6C 69 73 74 0D 00 00 00 00 00 00 00 2B  // Ключ - String; Данные - Список 
        1,                       05 00 01                                         // Подданые - UInt16
        0.1,                     08 CD CC CC 3D                                   // Подданые - Float
        false,                   0B 00                                            // Подданые - Boolean
        "Hello, World!",         0C 00 0D 48 65 6C 6C 6F 2C 20 57 6F 72 6C 64 21  // Подданые - String
        ["a", "b"]               0D 00 00 00 00 00 00 00 17 0C 00 01 61 0C 00 01  // Подданые - Список
                                 62
    ],
    "dict": {                    0C 00 04 64 69 63 74 0E 00 00 00 00 00 00 00 08  // Ключ - String; Данные - Словарь
        "a": "b"                 0C 00 01 61 0C 00 01 62                          // Подключ - String; Подданые - String
    },
    0: 0                         05 00 00 05 00 00                                // Ключ - UInt16; Данные - UInt16
}
```

Давайте сравним `BDSF`, `BSON` и `JSON`.
BDSF       | BSON        | JSON       | Разница
---------- | ----------- | ---------- | --------------
`151` байт | `175` байта | `132` байт | `24` и `-19` байт
