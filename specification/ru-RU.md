<h1>Спецификация <code>"Двоичного формата хранения данных" первой версии</code></h1>
<h2><strong>1.</strong> Формат содержимого файла</h2>
<h3><strong>1.1.</strong> Формат хранения данных</h3>
Все данные имеют следующий формат:

```py
SNTC

Где S - это "Сигнатура" пары ключ-значение состоящее из 1 байта со значением 0 (00 в HEX),
  а N - это значение типа 0A (String), которое обозначает ключ следующих данных,
  а T - это 1 байт типа данных,
  а C - это сами данные, записаные по формату, указаного в таблице.
  
# Например, число 5 записаное в UInt16 типе по ключу "num" будет выглядить так:
00 03 6E 75 6D 03 05 00 05
# И того 9 байта (3 байта - значение)

# А строка "Hello, World!" по ключу "str" будет выглядить так:
00 03 73 74 72 0C 00 0D 48 65 6C 6C 6F 2C 20 57 6F 72 6C 64 21
# И того 21 байт (16 байт - значение)
```

<h3><strong>1.2.</strong> Базовые типы</h3>
В этом пункте описаны самые базовые типы.

```
Тип данных ———————————————— Длина(Байты : Биты) ———— Диапозон ———————————————— Код типа —— Формат
Byte                        1        :        8      0      - 255              01          -
Int16                       2        :       16      -2¹⁵-1 - 2¹⁵+1            02          Стандартный "big" HEX формат
Int32                       4        :       32      -2³¹-1 - 2³¹+1            03          Стандартный "big" HEX формат
Int64                       8        :       64      -2⁶³-1 - 2⁶³+1            04          Стандартный "big" HEX формат
UInt16                      2        :       16      0      - 2¹⁶-1            05          Стандартный "big" HEX формат
UInt32                      4        :       32      0      - 2³²-1            06          Стандартный "big" HEX формат
UInt64                      8        :       64      0      - 2⁶⁴-1            07          Стандартный "big" HEX формат
Float                       4        :       32      1      - 7 a.             08          IEEE 754-2008
Double                      8        :       64      8      - 15 a.            09          IEEE 754-2008
Decimal128                  16       :      128      16     - 23 a.            0A          IEEE 754-2008
Boolean                     1        :        8      0      - 1                0B          C{1 : Значение 0 или 1 (Аналогия с C++: 0 = false; 1 = true)}
String                      ?        :        ?      -                         0C          L{2 : Длина кодированной строки ~Int16} C{L : Строка в UTF-8}

# a. - Знаки после точки
# Не описаные ранее форматы описываются в следующем виде: <char>{<len> : <description> [~<type>]}...
# : <...> - Обязательно; [...] - Не обязательно
# - char:        Символ, обозначающий описание байты
# - len:         Длина в байтах, может быть символом (см. char), так же может содержать N, который указывает на длину элементов в сумме
# - description: Описание байтов
# - type:        Ранее описанный тип
```
